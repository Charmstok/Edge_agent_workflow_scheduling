# Edge Agent Tool Offloading

An experimental framework for edge tool offloading and scheduling in multi-agent LLM systems.

This project studies how to schedule long-running tool calls generated by multiple LLM agents across heterogeneous edge devices. The focus is not on implementing OCR, PDF parsing, or table extraction algorithms from scratch. Instead, the project builds a standardized tool-call abstraction, worker execution interface, scheduling policies, execution tracing, and a reinforcement-learning optimization loop.

## Goal

The first implementation target is a minimal end-to-end execution loop:

```text
Simulated Agent -> Tool Call -> Tool Call Queue -> Baseline Scheduler
  -> Local Worker -> Tool Execution -> ToolResult -> Trace Log
```

## Environment

This project uses `uv` to manage Python environments and dependencies.

## Quick Start

Clone the repository and enter the project directory:

```bash
git clone <repo-url>
cd Edge-agent-tool-offloading
```

Install development dependencies:

```bash
uv sync --dev
```

Verify that the package can be imported:

```bash
uv run python -c "import edge_tool_offloading; print(edge_tool_offloading.__name__)"
```

Run tests:

```bash
uv run pytest
```

## Usage

The project is currently at the skeleton stage. The first runnable demo script will be added under `scripts/` once the core schemas, local worker, scheduler, and trace logger are implemented.

Expected command shape:

```bash
uv run python scripts/run_first_demo.py --policy round_robin
```

The demo will generate simulated tool calls, schedule them to local workers, execute a long-running tool, and write JSONL traces under `data/traces/`.

## Configuration

Configuration files live under `configs/`:

- `configs/system.yaml`: system-level settings
- `configs/devices.yaml`: worker device settings
- `configs/rl.yaml`: reinforcement-learning settings
